# Agent Team 适用判断矩阵

## 第一性原理

Agent Team 的价值来自**并发**和**职责隔离**。当这两者都不需要时，单 agent 更简单高效。

## 判断矩阵

| 条件 | 权重 | 评分 |
|------|------|------|
| 天然并发（≥2 独立工作单元） | 40% | 有=1, 无=0 |
| 长时运行（> 1h） | 30% | 是=1, 否=0 |
| 职责异构（发现/执行/修复/审计混合） | 20% | 是=1, 否=0 |
| 需要 push 消息（agent 间主动通信） | 10% | 是=1, 否=0 |

**加权得分 ≥ 0.6 → Agent Team；< 0.6 → 单 agent 或 subagents**

## 场景速查

| 场景 | 推荐 | 理由 |
|------|------|------|
| CS2 实盘（多场并发 + 16h） | Agent Team | 天然并发 + 长时运行 + 职责异构 |
| 参数回测（多组合并行） | Agent Team | 天然并发（参数组合爆炸） |
| PR 多角度审查 | Subagents | 并发但无 push 通信需求 |
| 单场比赛分析 | 单 Agent | 无并发需求 |
| 代码重构（单文件） | 单 Agent | 线性依赖，无并发 |
| 架构设计 | 单 Agent | 需要深度整体推理 |

## 不适合场景

- 任务线性（A→B→C 强依赖，无并发机会）
- 任务 < 30 分钟（spawn 开销超过收益）
- 共享可变状态密集（频繁读写同一文件）
- 深度推理设计类（拆分损失整体性）
