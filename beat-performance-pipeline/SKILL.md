# Beat 表演生产管线(分镜化 3D 角色表演)

## -1. 目标 / 最终验收标准 / 技术栈(评审与决策的基准,先读)

### 目标(一句话)
扫功夫茶冰箱贴 → 揭小贤完整演出【临现招手 → 走到茶桌 → 坐下 → 拿壶 → 六式冲茶(烫杯/纳茶/高冲/低斟/关公巡城/韩信点兵) → 敬你一杯】的 29.6s 表演,**与配音逐句同步**,AR 尺度下可爱、对版、物理可信、流畅 —— 让用户的体感是"**一个小向导刚刚请我喝了杯功夫茶**"。

### 最终验收标准(全部满足才算这一期完成)
1. **功能**:点「听讲解」后 B1–B14 全序列自动演完;每 beat 起止与配音 cue 误差 <0.3s;「再听一次」从 B3 重入且道具已复位。
2. **质量闸**:每个 beat 过 Tier A 全部不变量 + Tier B 五维(**判定规则:每维 双模型≥1 pass,且 ≥4/5 维 pass;双模型对同一维分歧→以"更严"一方的 issues 为修复输入,修后复评**);正面机位全程无可见穿模;所有道具拿/放无瞬移(re-parent 连续)。
3. **物理**:坐姿支撑(髋在凳上/脚踩地)、壶倾角规则(非倒茶<15°)、茶流重力(±5°)、端杯水平(<10°)全部机检通过。
4. **性能**:iPhone Safari 真机流畅(无 >1s 卡顿);本期新增 3D 资产合计 ≤3MB;模型可靠加载(此前 5.3MB 曾加载失败,3.2MB 稳)。
5. **体验(Tier C,唯一终审=用户)**:真机看整段"像小向导请你喝茶"——生动、可爱、不出戏;用户签字才封板。
6. **工程**:vitest 全绿;走 worktree+PR+merge;skill/契约文件同步更新。

### 技术栈选择(及为什么——都是验证过的结论,别再重选)
| 层 | 选型 | 为什么(替代方案为何不用) |
|---|---|---|
| 角色动画 | **Blender 程序化 IK+bake** | 已验证(Greet/Pour);Mixamo 要登录且 502 过;Tripo preset 与 mixamo-spec 骨架不兼容且无茶艺动作 |
| 走路位移 | clip 原地走 + runtime position lerp | 游戏标准技法;烘根位移会和锚定/重播打架 |

| 运行时/部署 | MindAR+A-Frame / GCE+cloudflare | 既有栈不动;iOS 非 meshopt+webp 铁律 |

## 0. 为什么用 Beat
一段 30s 连续表演没法整体评审/迭代;**Beat = 与配音 cue 逐句对齐的最小动作单元**(0.7–4s),
每个 beat 有自己的【物料/动作规格/可机检不变量/视觉评审点】→ 评审环才有抓手,失败只重做一个 beat。

## 1. 生产流程 v2(2026-06-12 全量复盘后重设计;旧"拿到什么先做什么"流程作废)
> 任务本质=一出微型舞台剧。**工程件自底向上(runtime/管线),内容件自顶向下(剧本→分镜→资产→舞台→表演)**——旧流程所有返工都源于把内容也自底向上做。

```
阶段0 立项   北极星+验收标准+工具能力实证清单(依赖的能力先真实成功调用一次才进计划)
阶段1 剧本   配音文案→TTS→cue表=唯一时钟
阶段2 分镜★  先出**空间设定包**(2a):场景平面图/侧视图(从坐标契约正交渲染,非画的)+人-桌-凳-茶具尺寸关系图
            +道具三视图(turnaround)——作为分镜与建模的**输入**;再(2b)codex 逐cue生成分镜
            (每格带全套茶桌茶盘,空间一致,有位移的格要画出移动轨迹/起止位)→评审→【用户批G0】
阶段3 资产★  从分镜推导道具清单→输入标准先行(对版参考/A-pose/无道具)→生成→压制gate
            →Tier V 视觉美感闸(§2.5)→资产总览用户过目;原点/bbox实测,不许拍脑袋
阶段4 舞台★  照分镜摆3D:原点修正+bbox进契约+可达性预验(臂长vs所有交互点)
            →"3D vs 分镜"并排图→【用户批G1】→契约冻结(冻结前禁止下游并行)
阶段5 表演   逐beat串行闸:codex设计→codex脚本(/tmp工件)→主循环Blender执行(每次干净导入)
            →Tier A(本地确定性,不依赖provider)→渲染(机位=分镜机位)
            →Tier B(基准=像不像分镜那格)→过闸入库;walk/sit优先动捕,程序化只做手势
阶段6 集成   白名单导出→资产链路检查表(新模型必须确认动画已重绑)→【无视觉证据不得部署】→真机=用户终审
```

### 1.5 验收矩阵(2026-06-12 G0-v3 批准后立;每步三层验收,缺一层=该步未完成)
| 步骤 | ① 自动脚本验收(确定性,本地) | ② 视觉验收 | ③ 代码验收 |
|---|---|---|---|
| 3a 道具参考图 | 文件存在>100KB;**轮廓比例机检**(PIL 前景 bbox,如桌 h/w∈[0.32,0.48]) | 主循环对版分镜+美感初筛 | — |
| 3b 建模+压制 | GLB 可 parse;bbox 三轴比例进容差;压后≤单件预算、总和≤3MB | 4 视角渲染→双模型 Tier V(对版/美感/压缩损伤/语义,基准=参考图+分镜) | 压制参数有出处 |
| 4 舞台 v3 | 接地误差<0.005;桌上物 y=桌面±0.005;bbox 互不穿插;出生点+路径点距桌 footprint≥0.05;全交互点 reach≤0.95×臂长 | 正交三视图+与 sb_S 对位并排图→双模型+主循环(G1') | 契约 diff 审:**每个数字写得出推导来源** |
| 5 动画重校 | 每 clip 自检 print PASS(脚穿/髋漂/触达/躯干上限/循环闭合) | (并入渲染步统一看) | 脚本 diff 审:只许坐标/时序变,逻辑不许漂 |
| 6 渲染+Tier B | 帧数=窗口×24±1;**角色入画机检**(前景 bbox 占画面高 25-70%);strip 拼装 exit 0 | 每 beat 单帧先审→全量;beats-visual-batch 双模型 5 维,**基准=对应分镜格**;主循环终筛 GIF | 机位表逐项=分镜镜头表 |
| 7 集成 | 导出 clip 清单=JYX_BEATS 引用集合(脚本读 GLB 比对);beat 窗口铺满 [0,29.6] 无缝隙无重叠(脚本验);build/tsc 0 错;资产总预算复核 | 本地 preview 逐 beat 截图证据包(=「无视觉证据不得部署」的证据) | runtime diff claude 关键位审 |
| 8 部署+终审 | 部署后 curl 200;线上资产 hash=本地 | **真机=用户终审** | — |

**横切纪律(根因→规则)**:
- 交付=验收物在手且过闸;"命令跑完/HTTP200/机检绿"都不算完成。
- 产出物校验闸:子任务空产出/只有开场白=FAILED 熔断上报,禁止静默当"没问题"下传。
- provider 评审撞 infra=阻断,绝不降级放行;核心不变量本地确定性化。
- 单变量迭代+两轮不涨熔断;一切渲染先单帧过目再全量。
- 角色分工:codex=生成产能(分镜/参考图/动作设计/脚本/取证);主循环=关键位(审实物/Blender执行/美术终筛/集成);用户=G0/G1/终审三个拍板点。
- 完整复盘报告(6根因+证据)见 session-forensics-v2 run(2026-06-12)。

## 2. 核心物料:坐标契约 scene-layout.json(一切空间关系的唯一真源)
```json
{ "units":"character_height=1.0", "anchor":"magnet center, x右 y上 z朝用户",
  "P0_spawn":[0,0,0.05], "table_pos":[...], "table_top_z":0.xx,
  "stool_pos":[...], "seat_hip":[...], "pot_on_table":[...],
  "cups":[[c1],[c2],[c3]], "gaiwan":[...], "walk_path":[[..],[..]] }
```
- **Blender 动画 authoring 与 runtime 布局读同一份契约** → 手伸向 `pot_on_table` 的 IK 目标 = runtime 壶的真实位置 = 手到壶到。
- 改布局=改契约文件,动画与场景同步变;**不许在代码/脚本里散落硬编码坐标**。
- ⚠️ **坐标系映射(已实证冻结,2026-06-11 B0)**:`blender = (c_z, c_x, c_y) × H + (0,0,z0)`,其中 H=角色 Blender 世界身高(实测 0.997≈1.0,契约单位≈Blender 米),z0=脚底。循环置换、行列式+1,相对几何全保。**推导曾给出错的 `−c_x`(把桌放反到角色右侧),靠"放标记物→渲染→对照 AR ry=270 真实朝向"才抓出来——映射永远实证,别信推导。**角色在 Blender 面朝 +X=契约 +z(朝用户);角色左手侧=+Y=契约 +x。runtime 侧契约直接用(A-Frame 锚系原生),无转换。
- **时序契约**:`audio.currentTime` 是唯一时钟;**beat 状态 = currentTime 的纯函数**(给定 t 能无状态推导出:当前 beat、walk 位移比例、壶/杯父子关系、茶流显隐)→ 暂停/seek/重播/锁屏中断天然正确。**禁止事件驱动累积状态**(中断后对不上)。重听=seek 到 7.5s(B3 起点)。

## 2.5 道具(资产)evaluator —— 任何道具进场景前必过(2026-06-12 被用户抓漏后强制化)
**字节/三角数 gate(Tier A)只是底线,不是验收。** 每个道具(桌/凳/壶/杯/盖碗/…)必须再过:
- **Tier V 视觉美感闸(双模型 workflow)**:Blender 渲 正面+3/4 两视图 → codex+claude 评 ①**对版**(潮汕红绿金描金、Q版潮玩质感,和揭小贤同一世界观?)②**美感**(造型/比例/贴图好看吗)③**压缩损伤**(狠减面后破相/穿洞/UV撕裂?)④**语义正确**(空茶盘就该是空的——Tripo 常夹带私货,如"桌上自带一把壶"与可抓壶打架,必须重生成而不是凑合)。
- **不过 → 重生成/换提示词/换来源,不许"先用着"。** 道具是每一帧都在镜头里的,丑一件毁全场。
- 流程:`Tripo 生成 → 压到 gate → 渲两视图 → 双模型视觉评审 → 过才进契约摆位`。

## 2.6 Beat 验收的三条铁律(同日补,防再犯)
1. **Tier B 视觉评审(beat-eval-loop.star)是强制步,不是可选**——"主循环自己瞄 filmstrip"只能当预筛,不算评审;每个 beat 必须跑双模型五维,记录在案。
2. **涉及位移的 beat(走路/移动)验收物必须带空间位移**:Blender 预览时给 armature 对象层加沿 walk_path 的位移动画(只为渲染,不烘进 clip),GIF 里看到"从 A 走到 B"——原地踏步的 GIF 验收不了"走到茶桌"。
3. **验收 GIF 一律上线 `public/beats/`**,用户看的是动图不是静帧。

## 3. 通用不变量(所有 beat 都检;**每条标注测量层** B=Blender scrub / R=runtime 探针 / V=视觉评审)
> **测量层原则(评审纠出的关键)**:Blender 只测得到骨骼/物体变换;**走路 lerp、re-parent 连续性、茶流 sprite、音频生命周期是 runtime 现象,必须用 R 层测**——做法:页面挂 `window.__jyxBeatProbe`(每帧记录 t/beat/关键世界坐标/倾角),headless Chromium(Playwright)跑一遍录下来→python 断言;V 层=渲染给视觉 agent/人。

- (B) 坐姿/站定期间 髋漂移≈0;**双脚 y≈地面**;躯干-桌沿距>0(不入桌)。
- (B) **可达性**:任何 IK 目标到肩的距离 ≤ (上臂+前臂)×0.95(Q版短臂,坐姿高冲/伸桌拿壶都要先验这条,不可达就改契约位)。
- (R) **道具 re-parent 帧世界变换连续**:快照世界矩阵→换父→反算 local;探针记录切换前后世界位置差 <1% 身高。
- (R) 走路防滑步:lerp 位移速度 = 步频×步长 ±10%。
- (B+R) **液体物理(统一阈值,单位一律【度】)**:出流阈值全局唯一 **>25°**(B 层检壶倾角轨迹,R 层检 sprite 显隐与倾角一致,滞回 ±3° 防闪烁);非倒茶时段壶倾角 **<15°**;端杯全程杯倾角 **<10°**;(R)茶流方向≈重力 ±5°,起点=壶嘴,终点≈正下方;高冲流长、低斟流短。
- (R) **茶汤状态机(从无到有)**:三杯液面(小圆片)= t 的纯函数——B9 巡城起出现、到 B10 末升至 70%("七分满");B13 端起的杯必须带茶面。简化声明:不模拟烫杯弃水,液面只进不退(AR 尺度够)。
- (R) **音频生命周期**:播放必须由用户手势启;锁屏/来电/切后台 → audio 暂停 → beat 纯函数自然冻结,恢复后从 currentTime 继续;「再听一次」= seek 7.5s 且道具状态由纯函数即时重算(无须手工复位)。
- (R) 道具拿起期间手-道具距恒定;表演结束(t>29.6)所有道具回契约初始位。
- 性能:**本期新增 3D 资产逐项预算**(总 ≤3MB):桌 ≤1.0MB(2.74 还要再压)/凳 ≤0.4/盖碗 ≤0.3/杯×3 ≤0.45(单杯 0.15)/壶 0.34(已达)/茶流+热气+液面 sprite ≤0.1;角色 GLB 增 4-6 个 clip 约 +0.05MB(clip 很小)。AR GLB 非 meshopt+webp。

## 5. 实现技法(已验证的,引用配套 skill)
- 动画 authoring:Blender IK 摆位→渲染迭代(我当眼)→`nla.bake`(visual_keying;Blender 5.x 用 `bpy.ops.pose.select_all` 不是 `bone.select`)→ ACTIONS 导出多 clip。见 `blender-mcp`。
- 走路位移:clip 原地走 + runtime 对 `a-gltf-model` position lerp(标准游戏技法)。
- re-parent:`bone.add(obj)`/快照世界矩阵;A-Frame 实体可被 three bone.add 收编(`jyx-handprop` 组件)。
- 口型=2D 说话脸(振幅包络),与 beat 并行不冲突;字幕=HTML 层。
- 道具来源:静物=Tripo text_to_model(28MB 原始→gltf-transform simplify 到 <1MB);对版 2D=codex 图生图。
